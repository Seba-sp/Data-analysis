**Create `reports/diagnosticos/generator.py`:**

Implement `DiagnosticosGenerator(BaseReportGenerator)` following the lifecycle contract from `BaseReportGenerator`. The generator assumes full-run semantics (no incremental_mode — Phase 4 will own that). Key implementation details:

**__init__:**
```python
from reports.base import BaseReportGenerator
from reports.diagnosticos.report_generator import ReportGenerator
from core.assessment_downloader import AssessmentDownloader
from core.assessment_analyzer import AssessmentAnalyzer
from core.storage import StorageClient
import os, logging
from pathlib import Path
from typing import Any, Dict, List
import pandas as pd

class DiagnosticosGenerator(BaseReportGenerator):
    ASSESSMENT_TYPES: List[str] = ["M1", "CL", "CIEN", "HYST"]

    def __init__(self):
        super().__init__("diagnosticos")
        self.downloader = AssessmentDownloader()
        self.analyzer   = AssessmentAnalyzer()
        self.report_generator = ReportGenerator()
        self.storage    = StorageClient()
        self._assessment_ids = {
            "M1":   os.getenv("M1_ASSESSMENT_ID"),
            "CL":   os.getenv("CL_ASSESSMENT_ID"),
            "CIEN": os.getenv("CIEN_ASSESSMENT_ID"),
            "HYST": os.getenv("HYST_ASSESSMENT_ID"),
        }
```

**download() → Dict[str, pd.DataFrame]:**
- For each assessment type in ASSESSMENT_TYPES, call `self.downloader.download_and_process_assessment(assessment_id, assessment_name, ...)` OR load from saved JSON via `load_responses_from_json` + `save_responses_to_csv`.
- Mirror the logic in `diagnosticos/main.py:download_all_assessments` and `process_all_assessments`.
- Save raw JSON to `self.raw_dir` and processed CSV to `self.processed_dir` via `self.downloader.save_responses_to_csv(responses, assessment_name)`.
- Return a dict mapping assessment type → processed DataFrame (or empty dict on failure).

**analyze(download_result) → Dict[str, pd.DataFrame]:**
- For each assessment type, call `self.analyzer.analyze_assessment_from_csv(assessment_name, question_bank_path, processed_csv_path, output_path, return_df=True)`.
- question_bank_path = `str(self.questions_dir / f"{atype}.csv")`
- processed_csv_path = path saved by download step under `self.processed_dir`
- output_path = `str(self.analysis_dir / f"{atype}.csv")`
- Return dict mapping assessment type → analysis DataFrame.
- Mirror the logic in `diagnosticos/main.py:analyze_assessment`.

**render(analysis_result) → Path:**
- Create output directory at `self.analysis_dir.parent / "output"` (i.e., `data/diagnosticos/output/`).
- For each assessment type and each row in the corresponding analysis DataFrame:
  - Build `user_info = {"username": row["email"], "email": row["email"]}`
  - Call `self.report_generator.generate_pdf(atype, row.to_dict(), user_info, incremental_mode=False, analysis_df=analysis_df)`
  - If pdf_content is not None, write bytes to `data/diagnosticos/output/informe_{email}_{atype}.pdf`
- Log a summary: N PDFs generated across M assessment types.
- Return `Path("data/diagnosticos/output")`.

**Error handling:** Wrap each assessment type's loop in try/except, log errors, continue to next type. Do NOT raise on per-student failure — matches standalone behavior.

**Important constraints (from CONTEXT.md):**
- `ASSESSMENT_TYPES = ["M1", "CL", "CIEN", "HYST"]` lives ONLY in `generator.py`. Do NOT add it to `core/` anywhere.
- Do NOT import or use `assessment_mapper.py` or `batch_processor.py` — those are GCP webhook artifacts.
- Do NOT implement incremental_mode — use full-run semantics only.
- Use `self.templates_dir`, `self.questions_dir`, `self.raw_dir`, `self.processed_dir`, `self.analysis_dir` from BaseReportGenerator (already wired for "diagnosticos").
