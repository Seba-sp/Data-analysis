   ```python
   def _make_schedule_timestamp(self, delay_seconds: int):
       ts = timestamp_pb2.Timestamp()
       ts.FromDatetime(
           datetime.datetime.now(tz=datetime.timezone.utc)
           + datetime.timedelta(seconds=delay_seconds)
       )
       return ts

   task = tasks_v2.Task(
       http_request=tasks_v2.HttpRequest(
           http_method=tasks_v2.HttpMethod.GET,
           url=f"{self.process_url}?report_type={report_type}&batch_id={batch_id}",
       ),
       schedule_time=self._make_schedule_timestamp(delay_seconds),
   )
   ```

4. Add `import datetime` at top.
5. Remove module-level `task_service = TaskService()` global instance.
6. Preserve delete_task, list_tasks, purge_queue, get_queue_info, create_queue_if_not_exists unchanged.
7. Add module docstring: "Cloud Tasks service for delayed batch processing callbacks."

<task type="auto">
  <name>Task 2: Promote BatchProcessor and AssessmentMapper to core/</name>
  <files>core/batch_processor.py, core/assessment_mapper.py</files>
  <action>
Create core/batch_processor.py. Base it on diagnosticos/batch_processor.py but replace the subprocess pattern with direct PipelineRunner invocation:

1. Imports:
   ```python
   from core.firestore_service import FirestoreService
   from core.runner import PipelineRunner
   ```
2. `__init__` accepts no arguments (report type is passed per-call, not at construction):
   ```python
   def __init__(self):
       self.batch_interval_minutes = int(os.getenv('BATCH_INTERVAL_MINUTES', '15'))
   ```
3. Remove `self.assessment_types` (hard-coded list) — assessment types are now an internal detail of each generator.
4. Replace `_process_assessment_type(assessment_type)` (which used subprocess) with `process_report_type(self, report_type: str) -> bool`:
   ```python
   def process_report_type(self, report_type: str) -> bool:
       try:
           runner = PipelineRunner(report_type=report_type)
           result = runner.run()
           logger.info(f"[{report_type}] Pipeline result: {result}")
           return result["success"]
       except Exception as exc:
           logger.error(f"[{report_type}] Pipeline error: {exc}")
           return False
   ```
5. `process_batch(self, report_type: str, batch_id: str) -> dict` — accepts report_type:
   - Gets queued students via `FirestoreService(report_type).get_queued_students()`
   - Calls `self.process_report_type(report_type)` once (not per-assessment-type)
   - Calls cleanup via `FirestoreService(report_type)` methods (clear_queue, clear_batch_state)
   - Returns `{success, batch_id, students_processed, processing_time, errors}`
6. Remove `_send_emails` method (not needed — PipelineRunner handles email).
7. Remove module-level `batch_processor = BatchProcessor()` global instance.
8. Add module docstring: "Batch processor that delegates report execution to PipelineRunner."
