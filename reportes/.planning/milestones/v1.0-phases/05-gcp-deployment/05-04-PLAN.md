  <name>Task 1: Build, push, and deploy to Cloud Run</name>
  <files></files>
  <action>
Run the GCP build and deploy sequence. The executor must have the GCP_PROJECT_ID env var set and gcloud authenticated.

Step 1 — build and push image:
```bash
gcloud builds submit --tag gcr.io/$GCP_PROJECT_ID/unified-pipeline .
```

Step 2 — get the PROCESS_BATCH_URL (needed before deploy if the service is being created fresh):
- If this is a first deploy, the URL is not yet known. Deploy without PROCESS_BATCH_URL first, then update the service with the URL after the first deploy completes.
- If updating an existing service, use the existing URL.

Step 3 — deploy webhook server:
```bash
gcloud run deploy unified-webhook \
    --image gcr.io/$GCP_PROJECT_ID/unified-pipeline \
    --platform managed \
    --region us-central1 \
    --memory 1Gi \
    --timeout 540 \
    --set-env-vars "GCP_PROJECT_ID=$GCP_PROJECT_ID,TASK_QUEUE_ID=batch-processing-queue,BATCH_INTERVAL_MINUTES=15,STORAGE_BACKEND=gcs" \
    --set-secrets "LEARNWORLDS_WEBHOOK_SECRET=learnworlds-webhook-secret:latest" \
    --allow-unauthenticated
```

Step 4 — update PROCESS_BATCH_URL once service URL is known:
```bash
SERVICE_URL=$(gcloud run services describe unified-webhook --region=us-central1 --format='value(status.url)')
gcloud run services update unified-webhook \
    --region=us-central1 \
    --set-env-vars "PROCESS_BATCH_URL=$SERVICE_URL/process-batch"
```

Step 5 — verify /status endpoint responds:
```bash
curl -s "$SERVICE_URL/status"
```
Expected: 200 JSON with `{"status": "healthy", "report_types": {"diagnosticos": {...}}}`

If any step fails with an auth error or missing env var, stop and surface the specific error — do not silently continue.
