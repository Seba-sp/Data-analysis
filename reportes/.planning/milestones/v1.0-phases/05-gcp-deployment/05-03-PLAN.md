  <action>
Create entrypoint.sh at repo root:

```bash
#!/bin/bash
# entrypoint.sh — mode switch for unified Cloud Run container
# If REPORT_TYPE is set: run pipeline as one-shot batch job and exit
# If REPORT_TYPE is not set: start webhook HTTP server (persistent)
set -e

if [ -n "$REPORT_TYPE" ]; then
    echo "[entrypoint] Batch mode: REPORT_TYPE=$REPORT_TYPE"
    exec python main.py --report-type "$REPORT_TYPE"
else
    echo "[entrypoint] Webhook server mode"
    exec functions-framework \
        --source=webhook_service.py \
        --target=webhook_handler \
        --port=8080
fi
```

Create Dockerfile at repo root:

```dockerfile
FROM python:3.11-slim

# WeasyPrint requires native Pango/HarfBuzz libraries (not pip-installable)
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpango-1.0-0 \
    libpangoft2-1.0-0 \
    libharfbuzz-subset0 \
    && rm -rf /var/lib/apt/lists/*

# Unbuffered output — important for Cloud Run log streaming
ENV PYTHONUNBUFFERED=1

WORKDIR /app

# Copy dependency manifest first — layer cache: pip install only reruns when requirements.txt changes
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code (do NOT copy data/ — Cloud Run uses STORAGE_BACKEND=gcs)
COPY core/ ./core/
COPY reports/ ./reports/
COPY templates/ ./templates/
COPY main.py webhook_service.py entrypoint.sh ./

RUN chmod +x entrypoint.sh

ENTRYPOINT ["/app/entrypoint.sh"]
```

Key decisions (per CONTEXT.md Claude's Discretion):
- Base image: `python:3.11-slim` — slim avoids alpine's wheel compatibility issues with pandas/numpy
- Layer order: requirements.txt before code — rebuild only reinstalls pip deps when code changes but deps don't
- ENTRYPOINT uses exec form array notation for proper signal propagation
- No COPY for data/, .env, diagnosticos/, diagnosticos_uim/ — these are excluded by design
- No .dockerignore needed for the excluded dirs since Dockerfile uses specific COPY commands; however, adding a .dockerignore for data/, .env, and __pycache__ is good practice if not already present — use discretion

Do NOT include a CMD instruction — entrypoint.sh handles both execution modes.
