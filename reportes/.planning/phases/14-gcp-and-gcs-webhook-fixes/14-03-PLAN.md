<objective>
Implement the per-assessment queue architecture. Currently, BatchProcessor downloads ALL
assessments for a report_type when any webhook fires. With this plan, each queued student
record carries an assessment_name set at webhook time, BatchProcessor groups by it, and
each generator downloads only the specific assessment that fired.

This prevents cross-assessment contamination (wrong student gets a report) and eliminates
failures caused by downloading assessments with invalid IDs when only one valid assessment
triggered the webhook.

Purpose: Correct end-to-end scoping so 40 concurrent webhooks each process independently.
Output: assessment_name flows from webhook -> Firestore -> BatchProcessor -> PipelineRunner -> generator.download().
</objective>
